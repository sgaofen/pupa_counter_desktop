# Agent Handoff — Pupa Counter Desktop (2026-04-30)

Supersedes / extends `HANDOFF_2026-04-23.md`. The previous doc described
the v12 + clf_v5 + min_distance=8 backend. This update is purely a
backend swap in the sibling `pupa_counter_v6` repo — **no code change
was needed in this Electron app**. This file just records the new
ship config so the next agent can read both handoffs and know what's
real.

## What changed today

In `pupa_counter_v6` (the Python worker we spawn):

| File | Edit |
|---|---|
| `pupa_counter.py` | `PEAK_MIN_DIST = 8 → 5` |
| `pupa_counter.py` | `CLASSIFIER_PATH = .../peak_filter_clf.pkl → .../peak_filter_clf_v6_md5.pkl` |
| `model/peak_filter_clf_v6_md5.pkl` | New 380 KB GBM classifier, retrained on V12's md=5 peak distribution |

The CNN backbone (`model/pupa_counter_v12.pt`) is unchanged. So the
desktop app spawning `pupa_counter_v6/pupa_counter_daemon.py` (or the
one-shot `pupa_counter.py`) automatically picks up the new pipeline
the next time the worker restarts. Nothing in `electron/main.js`,
`src/adapters/cnnAdapter.ts`, etc. needed to move.

## Numbers (full 99-scan / 10,145-label self-eval)

| Pipeline | F1 | Miss (edge / ctr) |
|---|---:|---:|
| v12 + clf_v5 + md=8 (legacy ship; backend before today) | 99.46 % | 110 (71 / 39) |
| v12 + clf_v6_md5 + md=5 (NEW backend, 2026-04-30) | **99.95 %** | 10 (6 / 4) |

Honest leave-one-scan-out CV: 99.08 % → 99.60 %, real-world delta
**+0.52 pp** (3.4 × fewer errors per pupa). See
`pupa_counter_v6/HANDOFF_2026-04-30.md` for the full derivation,
including the 10-miss anatomy and why the architectural floor is at
~4 interior misses.

## Smoke test

After the worker is restarted (or on first start), the daemon's
ready-message changes to:

```json
{"ready": true, "model": "pupa_counter_v12.pt", "classifier": "peak_filter_clf_v6_md5.pkl"}
```

If `classifier` still reads `peak_filter_clf.pkl`, your worker is
running off an older copy of `pupa_counter_v6` — pull and restart.

## Adapter / settings exposure

There is no UI surface for the inference parameters. They are
hard-coded in `pupa_counter_v6/pupa_counter.py`:

```python
PEAK_THRESHOLD = 0.15
PEAK_MIN_DIST = 5         # (was 8 before 2026-04-30)
CLASSIFIER_THRESHOLD = 0.60
```

If a future ticket wants Settings-tab controls for any of these,
the change is plumbing only — pass overrides via subprocess args
(both `pupa_counter.py` and `pupa_counter_daemon.py` already accept
positional / JSON-line args; threshold flags would be a small
addition).
