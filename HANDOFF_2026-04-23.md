## Elevator pitch

Electron desktop app that wraps the sister-repo
[`pupa_counter_v6`](https://github.com/sgaofen/pupa_counter_v6) (real
V12 CNN + classifier v5, F1 ≈ 99.46 %) into a lab tool Sarah / Anthony
use to scan pupae sheets, count them, hand-edit misses, and persist
everything to a local database.

All four app states were designed in Claude Design first, then ported
pixel-for-pixel into React. See `examples/` screenshots in the sister
repo for the look.

## Current status (2026-04-23)

### Fully wired & verified

- **Real CNN detection**: Electron main spawns ONE long-lived Python
  worker (`pupa_counter_daemon.py` in the v6 repo) on first request.
  Per-scan latency ≈ 0.75 s (vs ~3 s for subprocess-per-request).
- **Interactive EditCanvas** on the Scan view: pan (wheel / ⌃ + drag /
  arrows), zoom (wheel / + − / buttons), left-click add pupa,
  right-click delete nearest, ⌘Z undo, F fit, T top edge, B bottom
  edge. Edges can be panned to viewport centre so labels near the
  scanner black border stay clickable.
- **Per-pupa Database detail** panel with source (cnn / manual) and
  band colouring.
- **Session persistence** to `<userData>/session.json` with hydrate on
  app start and save-after-every-mutation.
- **Pending-scan round binding**: a scan started in round N commits
  to round N regardless of whether the user switches rounds later.
- **Mock fence**: real CNN failures throw (no silent fallback); the
  mock path only runs outside Electron OR with explicit
  `VITE_CNN_MOCK=1`. Mock detections are tagged and `Save to database`
  refuses them.

### Stubbed (intentionally — fill in tomorrow)

- **Scanner adapter** (`src/adapters/scannerAdapter.ts`) — opens a
  native file picker. Replace with real TWAIN (Win) / ICA (macOS)
  driver. See "Tomorrow's work" below for the shape of the fix.
- **TIFF support** — `file:readImageDataUrl` returns `image/*` for
  PNG/JPG/WebP. Scanners usually emit TIFF; add a one-pass conversion
  (e.g. `sharp`) in the main process.
- **SQLite** — currently using JSON. Swap `window.pupa.session.load /
  save` to `better-sqlite3` once the schema is frozen.

### Security / robustness (addressed in Codex adversarial review pass)

- Python `--json-out` now returns a structured per-image error record
  on `imread` failure instead of silently dropping inputs.
- `--json-out` refuses to overwrite any input image and writes via an
  atomic tmp+rename.
- `pupa_counter_daemon.py` emits a structured `{"ready": false,
  "stage": "startup", "error", "traceback"}` line before exiting if
  model/classifier load fails — the app can show a useful error
  instead of hanging.
- Electron CNN errors surface a red "Detection failed" pill and
  disable the Save button.
- Mock detections are explicitly tagged and blocked from ever
  reaching the database.

## How to clone & run on a fresh machine

### 0. Prerequisites

- **Node.js ≥ 20**, **npm ≥ 10**
- **Python 3.9+** with the `pupa_counter_v6` repo's deps installed
  (torch, torchvision, numpy, opencv-python, scikit-image, openpyxl,
  matplotlib). Easiest: clone the sister repo and `pip install -r
  requirements.txt` in a venv.

### 1. Clone both repos

```bash
git clone https://github.com/sgaofen/pupa_counter_desktop.git
git clone https://github.com/sgaofen/pupa_counter_v6.git
cd pupa_counter_desktop
```

### 2. Install renderer deps

```bash
npm install
```

### 3. Configure Python paths

The defaults in `electron/main.js` point at Stephen's personal paths.
On any other machine, either edit those constants or (preferably)
export environment variables before launching:

```bash
export PUPA_PYTHON="/absolute/path/to/venv/bin/python"
export PUPA_SCRIPT="/absolute/path/to/pupa_counter_v6/pupa_counter.py"
export PUPA_DAEMON="/absolute/path/to/pupa_counter_v6/pupa_counter_daemon.py"
```

Verify the daemon alone works before running the GUI:

```bash
echo '{"id":1,"cmd":"ping"}' | "$PUPA_PYTHON" "$PUPA_DAEMON"
# expected: first a ready line, then {"id":1,"ok":true,"result":"pong"}
```

### 4. Run in dev mode

```bash
npm run dev
```

Vite on :5173, Electron opens the window. The CNN worker starts on the
first scan request and stays alive for the rest of the session.

### 5. First sanity check

1. Click "Load demo scan" in the left sidebar. If this fails with "No
   demo scans found in pupate_batch", edit the `DEMO_DIR` constant in
   `electron/main.js` to point at a folder of real scans.
2. After ≈ 2 s the scan loads + detection runs + a green "Detection
   complete" pill should appear.
3. Right-click near a pupa dot → the dot should vanish (deleted).
4. Left-click an empty area → a blue dot (your addition) appears.
5. Click "Save to database" → toast "Saved s_XXXXX — N pupae".
6. Switch to the Database tab. The new scan should appear in the table.
   Click the row; the right panel shows per-pupa x / y / rank% / band /
   source.
7. ⌘Q the app, re-launch: the saved scan should still be there.

If any of these steps fail, read the Electron DevTools console
(Cmd+Option+I) for the real error.

## Tomorrow's work (scanner integration)

### Step 1 — Pick a driver stack

| Platform | API | Node lib | Notes |
|---|---|---|---|
| macOS | Image Capture Architecture (ICA) | `node-ffi-napi` bindings or shelling out to `icatop` | ICA is Apple-supported and ships with macOS. Avoid SANE unless truly needed. |
| Windows | TWAIN 2.x | `node-twain`, or a small C++ addon | TWAIN is universal, most lab scanners expose it. WIA is simpler but less common on scientific scanners. |

Stephen mentioned the scanner model in the Sarah email thread
("Pupa Counter" thread, 2026-04-20 onward) — inspect that and the
physical unit before committing to a lib.

### Step 2 — Wire scannerAdapter

Current interface (keep stable):

```ts
// src/adapters/scannerAdapter.ts
export interface ScanHandle {
  path: string;       // absolute path to the saved TIFF/PNG
  dataUrl: string;    // data: URL for immediate display in the UI
  width: number;
  height: number;
}
export async function scanNow(): Promise<ScanHandle | null>;
```

Replace the body of `scanNow()` with:

1. A new Electron IPC handler `scanner:scan` in `electron/main.js`.
2. That handler:
   - Calls the driver to acquire an image (TIFF buffer or file).
   - If TIFF, convert to PNG via `sharp` (add to deps).
   - Writes the PNG to the user's default save directory (configured
     on the Settings view, read from the store or
     `<userData>/settings.json`).
   - Returns the absolute path back to the renderer.
3. The renderer calls `window.pupa.file.readImageDataUrl(path)` to get
   the dataUrl for the canvas (already plumbed).

### Step 3 — Settings view wiring

The Settings → Scanner card already has a stubbed dropdown (shows "No
scanner connected"). Replace with a real device enumeration: call a
new `scanner:listDevices` IPC handler from the renderer on mount, and
store the chosen device in settings.

### Step 4 — Resolution, format, progress UI

- Acquisition usually takes ≥ 5 s at 1200 dpi. Show a progress
  indicator in the "Current scan" card during `scanNow()`.
- Consider a `scanner:cancel` IPC for aborting an in-progress scan.
- TIFF → PNG conversion via `sharp` is fast (~100 ms for our image
  sizes).

### Step 5 — Testing loop

1. With the scanner plugged in, `npm run dev`, click "New scan".
2. Watch the DevTools console + the Terminal running Vite for both JS
   and Python-side errors.
3. Confirm the full loop: acquire → detect → display → edit → save →
   reopen app → scan still there.

## Data model (frozen — don't break)

```
Session
 ├── sessionId, operator, experiment, startedAt, endedAt?
 └── rounds: Round[]
      ├── roundId, roundNumber, startedAt, notes?
      └── scans: ScanRecord[]
           ├── id, roundNumber, imageNumber, timestamp, imagePath
           ├── imageWidth, imageHeight
           ├── experiment, operator, genotype, comments, infoFilename
           ├── totalPupae, top5PctCount, rank5To25Count, middle50Count,
           │   bottom25Count, yMin, yMax
           ├── manuallyEdited: boolean
           └── pupae: Pupa[]
                ├── index (1-based)
                ├── x, y (pixels)
                ├── rankPct (0=image bottom … 100=image top)
                ├── band ('0-5%' | '5-25%' | '25-75%' | '75-100%')
                └── source ('cnn' | 'manual')
```

SQLite schema (when you swap over) is in the chat history —
roughly `sessions → rounds → scans → pupae` with FK relationships.
Import-export formats must match this shape.

## Known issues / deferred work

1. **Manual edit keyboard focus**: ⌘Z and arrow panning work only when
   the canvas has focus (hover suffices). Formal focus ring TBD.
2. **Session "Start new round" has no undo**. Consider a confirm dialog
   or allow merging empty rounds on exit.
3. **No autosave indicator**. Every mutation triggers a background
   save but the user doesn't see it — add a subtle checkmark.
4. **Database filters are decorative** — the chips render but don't
   actually filter the table.
5. **No multi-select / bulk delete** in Database view.
6. **Column sort** in Database table is marked with a sort indicator
   but isn't wired.

None of these block the happy path — they're polish for after scanner
integration is stable.
